# 阶段性总结（作品集快照）

我用“证据优先”的方式复盘这段工作：哪些结论我已经做成可审查证据，哪些目标我明确还没达成。这里不使用内部命名，只用通用口径。

---

## 0. 结论

这些都能在本仓库证据样例里直接找到对应字段：

- **压缩倍率与质量**：样例里 `ratio_fp16_effective_min = 4.0` 且 `cos_min = 0.999954`，代表我在当前阶段能做到“倍率提升的同时，保持极高相似度”。  
  - 直观换算：**4.0x ≈ 体积减少 75%**（\(1 - 1/4.0 = 75\%\)）。
- **可复现性**：样例里 `determinism_delta = 0.0`，表示我把“可复现性”做成可被机器验证的字段，而不是一句话承诺。
- **可解释性**：样例里 `bytes_total/bytes_field/bytes_patch/bytes_entropy` 把“字节口径”拆开，能直接看到元数据/补丁/熵相关开销的量级（避免只报一个总数）。

---

## 1. 证据链总览

- **门禁**：我用一套统一门禁读取证据 JSON，按阈值给出 GO/NO-GO，并把报告落盘，方便复盘与对外沟通。
- **运行时内存**：推理峰值内存门槛已 **PASS**（<1024MB），可以复现“<1GB 峰值内存”这一类结论。
- **静态压缩**：当前倍率约 **4.7x**、对齐约 **0.99**；目标倍率（20x）未达。我认为目前瓶颈在编码/变换域，而不是“对齐指标写法”。
- **分阶段代理评估**：我用三阶段代理指标把压缩评估做成可复盘证据；相似度可以做到很高（≈0.9999 的量级），并接入了可逆预处理与可选的 PPL/对齐辅助评估。
- **可复现性/口径**：我提供了 determinism 与 bytes 口径的证据字段，便于判断“是否漂移”“字节统计是否一致”。

---

## 2. 静态压缩：已摸清的 Pareto

- **高倍率区**：可到 ~9.6x，但 cos_min 约 0.94（对齐不足）。
- **高对齐区**：cos_min≥0.99 时，最佳倍率约 ~5.65x（与当前 4–6x 平台一致）。
- **结论**：在不更换坐标系、不做二阶自适应变换的前提下，高倍率与高对齐不可兼得；下一阶段需在「最优坐标系」下再做量化与倍率攻关。
